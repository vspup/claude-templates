# Claude Templates

Личная коллекция шаблонов и воркфлоу для работы с [Claude Code](https://docs.claude.com/en/docs/claude-code/overview).

**Репозиторий:** https://github.com/vspup/claude-templates
**Languages:** [English](./README.md) | Русский

---

## Содержимое

- [`project-bootstrap.md`](./project-bootstrap.md) — спецификация для развёртки нового проекта с поддержкой Claude Code (каноническая версия, EN)
  - [`project-bootstrap.ru.md`](./project-bootstrap.ru.md) — русский перевод

---

## Как использовать

1. Обсуди с ИИ идею проекта, выбери стек, сделай первоначальный анализ
2. В конце обсуждения прикрепи этот файл и попроси:
   > "Сгенерируй bash-скрипт развёртки проекта по этому шаблону с учётом нашего обсуждения"
3. Запусти полученный `bootstrap.sh`
4. `cd <project> && claude` — работай

## Как хранить и версионировать

- **Храни шаблон отдельно от проектов** — в личном репозитории (например, `<you>/claude-templates` или `<you>/dotfiles`)
- **Версионируй через git-теги** — `v1.0`, `v1.1`, чтобы можно было ссылаться на конкретную версию
- **НЕ копируй этот файл в `docs/` проекта** — вместо этого фиксируй применение в `docs/decisions.md` со ссылкой на версию
- **Эволюционируй** — дописывай что работает на практике, удаляй что не работает

### Способы подключения

Три варианта использования шаблона из проекта. Выбери один:

1. **Внешняя ссылка (по умолчанию).** Шаблон живёт в отдельном репозитории. `docs/decisions.md` фиксирует версию, использованную при bootstrap. Никаких файлов шаблона внутри проекта. Самый простой вариант, рекомендуется в большинстве случаев.

2. **Git submodule.** Подключи шаблон как submodule, например в `.claude/templates/`. Полезно если хочешь зафиксировать конкретный коммит и обновляться осознанно. Компромисс: добавляет механику submodule в ежедневные git-операции.
   ```
   git submodule add https://github.com/vspup/claude-templates.git .claude/templates
   ```

3. **Снапшот в docs/meta/.** Скопируй файл шаблона в `docs/meta/bootstrap-snapshot.md` в момент bootstrap. Используй только в airgapped-средах без доступа к внешнему репозиторию. Добавь `docs/meta/` в `.claudeignore`.

По умолчанию — вариант (1). Переходи к (2) или (3) только при конкретной причине.

## Правило языка

Репозиторий следует правилу, которое сам и продвигает:

- **Канонические файлы:** английский
- **Переводы:** русский, с суффиксом `.ru.md`

При расхождениях между английской и русской версиями — английская имеет приоритет.

Почему: английский даёт портируемость и соответствие индустриальным стандартам. Русские переводы — для удобства работы в русскоязычном контексте.

## Версионирование

Semantic Versioning через git-теги: `v1.0`, `v1.1`, `v1.2`, ...

**При ссылке на шаблон из проекта** используй URL с тегом:

```
https://github.com/vspup/claude-templates/blob/v1.4/project-bootstrap.md
```

Тогда `docs/decisions.md` проекта будет всегда указывать на стабильный снапшот, даже если шаблон потом эволюционирует.

История версий — в [`CHANGELOG.md`](./CHANGELOG.md).
