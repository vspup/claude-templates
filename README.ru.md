# Claude Templates

Личная коллекция шаблонов и воркфлоу для работы с [Claude Code](https://docs.claude.com/en/docs/claude-code/overview).

**Languages:** [English](./README.md) | Русский

---

## Содержимое

- [`project-bootstrap.md`](./project-bootstrap.md) — спецификация для развёртки нового проекта с поддержкой Claude Code (каноническая версия, EN)
  - [`project-bootstrap.ru.md`](./project-bootstrap.ru.md) — русский перевод

---

## Как пользоваться шаблоном

1. Обсуди с ИИ идею проекта, выбери стек, сделай первоначальный анализ
2. Прикрепи нужный файл шаблона в чат
3. Попроси ИИ сгенерировать bootstrap-скрипт на основе шаблона и обсуждения
4. Запусти скрипт, начинай работу

Детали workflow — в каждом конкретном файле шаблона.

---

## Правило языка

Репозиторий следует правилу, которое сам и продвигает:

- **Канонические файлы:** английский
- **Переводы:** русский, с суффиксом `.ru.md`

При расхождениях между английской и русской версиями — английская имеет приоритет.

Почему: английский даёт портируемость и соответствие индустриальным стандартам. Русские переводы — для удобства работы в русскоязычном контексте.

## Версионирование

Semantic Versioning через git-теги: `v1.0`, `v1.1`, `v1.2`, ...

**При ссылке на шаблон из проекта** используй URL с тегом:

```
https://github.com/<you>/claude-templates/blob/v1.2/project-bootstrap.md
```

Тогда `docs/decisions.md` проекта будет всегда указывать на стабильный снапшот, даже если шаблон потом эволюционирует.

История версий — в [`CHANGELOG.md`](./CHANGELOG.md).
