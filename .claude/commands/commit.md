Создай коммит по текущим изменениям в репозитории шаблонов.

Шаги:
1. Запусти `git status` и `git diff --staged`
2. Проверь специфику этого репо:
   - Если изменения касаются шаблонов (`project-bootstrap*.md`) — убедись, что EN и RU версии синхронизированы
   - Если менялась версия в заголовках — убедись, что она обновлена в обоих файлах и в CHANGELOG.md
   - Если это изменение в шаблоне — должна быть запись в CHANGELOG.md с секцией "Why"
3. Сгенерируй сообщение на английском в стиле Conventional Commits:
   - `feat:` новые принципы, секции, поддержка стеков в шаблоне
   - `fix:` исправления существующего содержимого шаблона
   - `docs:` правки README / CHANGELOG, не касающиеся самих шаблонов
   - `chore:` мелочи (gitignore, license, форматирование)
4. Покажи сообщение ПЕРЕД коммитом
5. После моего подтверждения — `git commit`

Если меняется версия — напомни мне после коммита поставить git tag:
`git tag v<X.Y> && git push --tags`
